About the experiment

As an improvement over the normal way of storing user session, $cookieStorage was used. In general the normal js session is wiped out when the broweser is refreshed, but cookie storage ensures persistent session storage in this case. $cookieStrorage Provides a key-value (string-object) storage, that is backed by session cookies. Objects put or retrieved from this storage are automatically serialized or deserialized by angular's toJson/fromJson.
In this case, logging in using the credentials "user" and "password" maintains a session for the user and shows the log out button
