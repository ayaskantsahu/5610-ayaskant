About the experiment

You will need to register and sign in to view the home page

Angular js arouting has been used to make the most of partials. This enables loading of html chunks withing a page without refreshing the entire page. Angular route is used for this purpose. In my case, the various links on the menu lead to different pages. Angular routing has been used to achieve this.


Code Snippets

These lines define the links on the html which will invoke angular routing upon interaction. 
 
            <li>
                <a href="/viewTimeRange">Time based</a>
            </li>
            <li>
                <a href="/viewStaff">User based</a>
            </li>
            <li>
                <a href="/viewAll">All Staff</a>
            </li>


These lines are the routes configured in angular js. They help loading of partials and also specify which controller will handle the new partial loaded and its data. 
         $routeProvider
		.when('/assignRole', {
			templateUrl: 'views/AssignRole.html',
			controller: 'AssignRoleController'
		})
		.when('/viewAll', {
			templateUrl: 'views/AllStaffView.html',
			controller: 'AllStaffController'	
		})
		.when('/viewStaff', {
			templateUrl: 'views/SingleUserView.html',
			controller: 'SingleUserController'	
		})
		.when('/dashboard', {
			templateUrl: 'views/Dashboard.html',
			controller: 'DashboardController'	
		})
		.when('/viewTimeRange', {
			templateUrl: 'views/TimeRangeView.html',
			controller: 'TimeRangeController'	
		})
		.otherwise({
			redirectTo: '/dashboard'
		});
